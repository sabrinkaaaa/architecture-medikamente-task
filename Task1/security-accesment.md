# Анализ безопасности системы "Медикаменте"

## 1. Конфиденциальные данные

### Персональные данные пациентов
- Ф.И.О.
- Дата рождения
- Телефон
- Электронная почта
- Адрес прописки
- Место работы/учёбы

### Медицинские данные
- Медицинские карты
- Результаты анализов
- Заключения врачей
- История болезни
- Диагнозы

### Финансовые данные
- Данные платежей
- Финансовые отчёты
- Бухгалтерские документы
- Данные ККМ

### Данные сотрудников
- Кадровые данные
- Зарплатные ведомости
- Данные доступа к системам

## 2. Аудит мер по обеспечению безопасности данных

### Проблемы безопасности

#### 2.1 Хранение данных
- **Проблема**: Данные хранятся в неструктурированном виде (Excel, JPG, PDF) на общем диске
- **Риск**: Высокий - несанкционированный доступ, утечка данных
- **Соответствие требованиям**: Не соответствует требованиям 152-ФЗ

#### 2.2 Контроль доступа
- **Проблема**: Отсутствует разграничение доступа к конфиденциальным данным
- **Риск**: Высокий - любой сотрудник может получить доступ к любым данным
- **Соответствие требованиям**: Не соответствует принципам минимальных привилегий

#### 2.3 Аудит действий
- **Проблема**: Отсутствует аудит действий с данными
- **Риск**: Средний - невозможность отследить утечки и нарушения
- **Соответствие требованиям**: Не соответствует требованиям по аудиту

#### 2.4 Шифрование данных
- **Проблема**: Данные не шифруются при хранении и передаче
- **Риск**: Высокий - данные могут быть перехвачены или украдены
- **Соответствие требованиям**: Не соответствует современным стандартам безопасности

#### 2.5 Резервное копирование
- **Проблема**: Отсутствует система резервного копирования
- **Риск**: Высокий - потеря данных при сбоях
- **Соответствие требованиям**: Не соответствует требованиям по обеспечению доступности

#### 2.6 Интеграции
- **Проблема**: Отсутствует контроль за передачей данных между системами
- **Риск**: Средний - утечка данных через интеграции
- **Соответствие требованиям**: Не соответствует принципам Privacy by Design

## 3. Рекомендации по улучшению

### 3.1 Классификация данных и способы защиты

| Категория данных | Способы защиты | Приоритет |
|------------------|----------------|-----------|
| Персональные данные пациентов | Шифрование, обезличивание, RBAC | Высокий |
| Медицинские данные | Шифрование, обфускация, аудит доступа | Критический |
| Финансовые данные | Шифрование, цифровые подписи, RBAC | Высокий |
| Данные сотрудников | Шифрование, RBAC, аудит | Средний |

### 3.2 Механизм тегирования данных

#### Теги по уровню конфиденциальности:
- **PUBLIC** - общедоступные данные
- **INTERNAL** - внутренние данные компании
- **CONFIDENTIAL** - конфиденциальные данные
- **RESTRICTED** - данные ограниченного доступа
- **MEDICAL** - медицинские данные (особая категория)

#### Теги по типу данных:
- **PII** - персональные данные
- **FINANCIAL** - финансовые данные
- **MEDICAL** - медицинские данные
- **HR** - кадровые данные

### 3.3 Инструменты и меры защиты

#### Технические меры:
1. **Шифрование данных**:
   - AES-256 для данных в покое
   - TLS 1.3 для данных в передаче
   - Управление ключами шифрования

2. **Контроль доступа**:
   - RBAC (Role-Based Access Control)
   - ABAC (Attribute-Based Access Control)
   - Многофакторная аутентификация

3. **Мониторинг и аудит**:
   - Логирование всех действий с данными
   - Система обнаружения аномалий
   - Алерты при подозрительной активности

4. **Резервное копирование**:
   - Автоматическое резервное копирование
   - Шифрование резервных копий
   - Тестирование восстановления

#### Организационные меры:
1. **Политики безопасности**:
   - Политика обработки персональных данных
   - Политика информационной безопасности
   - Регламенты работы с конфиденциальными данными

2. **Обучение персонала**:
   - Обучение основам информационной безопасности
   - Регулярные тренинги по работе с данными
   - Тестирование знаний

3. **Управление инцидентами**:
   - План реагирования на утечки данных
   - Процедуры уведомления регуляторов
   - Анализ инцидентов и улучшение процессов
